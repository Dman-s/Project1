# 登录加载聊天记录 & 退出重置为空白状态

## Context（背景）

用户反馈:
1. **登录后**左侧栏看不到自己的聊天历史记录（仍显示初始空白会话）。
2. **退出登录后**左侧栏仍保留上一个账号加载的聊天记录，未恢复到"刚打开软件时的空白状态"。

根因分析（均在 [useStore.js](file:///d:/pro/Project1/frontend/src/store/useStore.js)）：

- **`login` 函数（L155-181）**：设置 token 和 user 后，**没有调用 `loadConversations()`**。而 `getCurrentUser`（L188-213）在刷新页面恢复登录态时，于 L203 正确调用了 `loadConversations()`。因此"刷新页面能看到记录、但点登录按钮看不到"。
- **`logout` 函数（L183-186）**：只 `removeToken()` 和 `set({ user: null })`，**没有重置 `conversations` / `activeConversationId`**。所以退出后侧栏仍残留上一个用户的历史。
- 初始"刚打开软件"的空白状态定义在 L117-126：一个 id 为 `'1'`、标题 `New Conversation`、空消息的占位会话。

## 实现方案

仅修改 [frontend/src/store/useStore.js](file:///d:/pro/Project1/frontend/src/store/useStore.js) 一个文件，两处改动：

### 改动 1：`login` 登录成功后加载用户聊天记录

在 `login` 函数（L155-181）中，`set({ user: {...} })` 之后、`return data` 之前，追加：

```js
await get().loadConversations()
```

与 `getCurrentUser`（L203）保持一致。`loadConversations`（L215-234）已正确实现：后端返回记录则替换并激活首条；返回为空则保留占位空白会话（即"无历史记录"也显示空白，符合预期）。

### 改动 2：`logout` 退出时重置为初始空白状态

将 `logout` 函数（L183-186）由：

```js
logout: () => {
  removeToken()
  set({ user: null })
},
```

改为：

```js
logout: () => {
  removeToken()
  set({
    user: null,
    conversations: [
      {
        id: '1',
        title: 'New Conversation',
        messages: [],
        createdAt: new Date(),
      },
    ],
    activeConversationId: '1',
  })
},
```

重置内容与 L118-126 的初始状态完全一致，即"刚打开软件时的空白聊天记录"。`user: null` 使 Home.jsx 顶栏恢复 Login 按钮。

## 涉及文件

- [frontend/src/store/useStore.js](file:///d:/pro/Project1/frontend/src/store/useStore.js)（仅此一个文件，两处编辑）

无需改动 Home.jsx、LeftSidebar.jsx、LoginModal.jsx：登录入口统一走 `login()`，退出入口统一走 `logout()`（Home.jsx L174），侧栏直接渲染 `conversations`（LeftSidebar.jsx L62），状态重置后 UI 自动响应。

## 验证方式

1. 启动前端：`cd frontend && npm run dev`，启动后端。
2. **登录场景**：用已有账号登录 → 左侧栏应显示该账号在后台的聊天记录列表，并默认激活第一条。
3. **退出场景**：点击头像菜单"退出账户" → 左侧栏恢复为单条空白 `New Conversation`，顶栏出现 Login 按钮，不残留任何上一个账号的记录。
4. **新用户场景**：注册新账号登录（后端无历史记录）→ 左侧栏显示空白 `New Conversation`（占位会话），符合"无历史则空白"。
5. 刷新页面保持登录态 → 仍正常加载记录（`getCurrentUser` 路径，回归验证未受影响）。
